JDBC 환경 설정

기본 환경
  • JDK 1.5 이상
  • CUBRID 2008 R1.0 이상
  • CUBRID JDBC Driver 2008 R1.0 이상
Java 설치

시스템에 Java가 설치되어 있고 JAVA_HOME 환경 변수가 등록되어 있어야 한다. Java는 Sun MicroSystems의 Java 홈페이지(http://java.sun.com)에서 다운로드하여 설치할 수 있다. Java 설치 후 설치된 Path를 JAVA_HOME 환경 변수로 추가하고, PATH 환경 변수에 %JAVA_HOME%\bin을 추가한다.

JAVA_HOME 환경변수는 아래의 그림과 같이 [내 컴퓨터] > [속성] > [고급] > [환경 변수] > [시스템 변수 새로 만들기] > [새 시스템 변수]에 추가한다.

 


PATH 환경 변수는 아래 그림과 같이 [내 컴퓨터] > [속성] > [고급] > [환경 변수] > [시스템 변수 Path 선택] > [편집] > [시스템 변수 편집]에서 수정한다.

Linux에서는 사용하는 셸의 종류에 따라 JAVA_HOME과 PATH를 설정하여 적용한다(예: sh | bash).

export JAVA_HOME=<JAVA가 설치된 Full Path>
export PATH=$JAVA_HOME/bin:$PATH

CLASSPATH 설정

JDBC를 사용하려면 CUBRID JDBC 드라이버가 존재하는 경로를 환경 변수 CLASSPATH에 추가해야 한다.
CUBRID JDBC 드라이버는 Linux와 Unix의 경우 $CUBRID/jdbc에, Windows의 경우 %CUBRID%\jdbc에 위치한다.

 

Windows에서는 다음과 같은 CLASSPATH를 JAVA_HOME 환경 변수를 등록했던 방법과 동일한 방법으로 등록한다.

CLASSPATH=C:\CUBRID\jdbc\cubrid_jdbc.jar:.

Linux에서는 사용 셸에 따라 다음과 같은 형식으로 CLASSPATH를 등록하고 반영한다(예: sh | bash).

export CLASSPATH=$HOME/CUBRID/jdbc/cubrid_jdbc.jar:.

만약 JRE가 설치된 라이브러리 디렉터리($JAVA_HOME/jre/lib/ext)에 일반 CUBRID JDBC 드라이버가 설치되어 있다면, Java 저장 프로시저에서 사용하는 서버 사이드 JDBC 드라이버보다 먼저 로딩되어 Java 저장 프로시저가 비정상적으로 구동될 수 있다. Java 저장 프로시저를 사용하는 환경에서는 JRE가 설치된 라이브러리 디렉터리($JAVA_HOME/jre/lib/ext)에 일반 CUBRID JDBC 드라이버를 설치하지 않도록 주의한다.
