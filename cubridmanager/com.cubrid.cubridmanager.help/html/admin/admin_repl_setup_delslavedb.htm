슬레이브 데이터베이스 삭제

슬레이브 데이터베이스를 완전히 삭제하기 위해서는 복제 에이전트를 중지한 후 배포 DB를 삭제한다.

  • (단계1) 복제에이전트 중지
  • cubrid repl_agent stop을 사용하며 배포DB명을 파라미터로 지정한다.

cubrid repl_agent stop sample_dist

  • (단계2) 배포DB 삭제
  • 배포DB 서버를 중지하고 배포DB를 삭제한다.

cubrid server stop sample_dist
cubrid deletedb sample_dist

  • (단계3) 복제 관련 로그 삭제
  • 복제 수행시 만들어진 <master_db_name>.copy, <master_db_name>.copy.ar0, <dist_db_name>.trail, <dist_db_name>.err 파일을 삭제한다.

만약, 일시적으로 복제 작업을 중단하는 경우에는 복제 에이전트와 복제 서버만 중지하고 추후 복제를 다시 시작할 때 복제 서버를 먼저 시작한 후 복제 에이전트를 시작한다.
