데이터베이스 트랜잭션 제거

cubrid killtran은 옵션에 따라 대상 데이터베이스에서 원하는 트랜잭션을 제거하는 유틸리티이다.

구문

cubrid killtran options  database_name
options : [{-i|--kill-transaction-index=}index]  [--kill-user-name=id]  [--kill-host-name=host]
             [--kill-program-name=name]  [--kill-program-name=name] [{-p|--dba-password=}password]
             [-d|--display-information]  [-f|--force]

  • cubrid : CUBRID 서비스 및 데이터베이스 관리를 위한 통합 유틸리티이다.
  • killtran : 대상 데이터베이스에서 옵션에 따라 트랜잭션을 제거한다.
  • options : 제거하고자 하는 트랜잭션 대상을 지정하는데 사용한다. 옵션이 지정되지 않으면 해당 데이터베이스의 모든 트랜잭션을 대상으로 한다. -p 옵션 뒤에 오는 값은 DBA의 암호이며 생략하면 프롬프트에서 입력해야 한다.
  • database_name : 제거하고자 하는 트랜잭션의 데이터베이스의 이름이다.
옵션
  • testdb 데이터베이스의 모든 트랜잭션 보기

cubrid killtran testdb

  • 지정한 인덱스에 걸려있는 트랜잭션 제거(-i 또는 --kill-transation-index)

cubrid killtran -i test_index testdb

  • 지정한 사용자ID에 걸려있는 트랜잭션 제거(--kill-user-name)

cubrid killtran --kill-user-name myuser testdb

  • 지정한 클라이언트 호스트에 걸려있는 트랜잭션 제거(--kill- host-name)

cubrid killtran --kill-host-name myhost testdb

  • 지정한 프로그램에 걸려있는 트랜잭션 제거(--kill-program-name)

cubrid killtran --kill-program-name my_test_program testdb

  • 액티브 트랜잭션에 대한 정보 출력(-d 또는 --display)

cubrid killtran -d testdb

  • 중지할 트랜잭션을 확인하는 프롬프트 생략(-f 또는 --force)

cubrid killtran -f -i 1 testdb
