서비스 상태 관리

등록된 마스터 프로세스, 데이터베이스 서버 및 복제 서버의 상태를 확인하기 위하여 다음과 같이 입력한다.

% $ cubrid service status
@ cubrid master status
++ cubrid master is running.
@ cubrid server status

 Server testdb (rel 8.2, pid 31059)
 Server demodb (rel 8.2, pid 30950)

@ cubrid broker status

NAME            PID    PORT    AS    JQ  REQ  TPS  OPS  AUTO    SES    SQLL    CONN
=====================================================================================
* query_editor  336   30000     5      0  0   ---  ---   ON     OFF     ON:A AUTO
* broker1       5100  33000     5      0  0   ---  ---   ON     OFF     ON:A AUTO

@ cubrid manager server status
++ cubrid manager server is not running.
@ cubrid replication status

만약, 마스터 프로세스가 중지된 상태라면, 다음과 같은 메시지가 출력된다.

% cubrid service status
@ cubrid master status
++ cubrid master is not running.
