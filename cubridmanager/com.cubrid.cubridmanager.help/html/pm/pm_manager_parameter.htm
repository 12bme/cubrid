파라미터 설명

cm_port

cm_port는 CUBRID 매니저 서버와 매니저 클라이언트 사이에 이용되는 포트를 설정하기 위한 파라미터이다. 실제 사용되는 포트는 cm_port 파라미터에 의해 설정된 포트와 포트 번호에 1을 더한 포트 두 개가 사용된다. 예를 들어, 디폴트 값인 8001이 지정된 경우, 실제 이용되는 포트 번호는 8001, 8002이다. cm_port는 cub_auto가 사용하며, cm_port+1은 cub_js가 사용한다.

monitor_interval

monitor_interval은 CUBRID 매니저 서버의 cub_auto 프로세스에 대한 모니터링 주기를 초 단위로 설정하는 파라미터로 디폴트 값은 5(sec)이고, 최소값은 1(sec)이다. monitor_interval이 짧을수록 시스템 부하가 커진다.

allow_user_multi_connection

allow_user_multi_connection은 CUBRID 매니저 서버에 여러 개의 매니저 클라이언트가 접속할 수 있도록 사용자의 중복 접속을 허용하는 파라미터이다. 디폴트 값인 YES로 설정하면 사용자는 동일한 사용자 이름을 가지고 두 개 이상의 CUBRID 매니저 클라이언트에서 하나의 매니저 서버에 접속할 수 있다.

auto_start_broker

auto_start_broker는 CUBRID 매니저 서버가 구동될 때 CUBRID 브로커도 자동으로 구동되도록 설정하는 파라미터이다. 디폴트 값인 YES로 설정하면 CUBRID 매니저 서버와 함께 CUBRID 브로커도 구동된다.

execute_diag

execute_diag은 CUBRID 매니저에서 제공되는 데이터베이스에 대한 진단 기능을 활성화하는 파라미터이다. 디폴트 값인 OFF로 설정하면 데이터베이스의 진단 기능이 비활성화되고, ON으로 설정하면 CUBRID 매니저를 통해 데이터베이스의 진단 기능을 활용할 수 있다. 다만, 이 파라미터는 데이터베이스 진단 기능에 관한 것이므로, 브로커의 진단 기능은 이 파라미터의 설정값과 관계없이 활성화된다.

server_long_query_time

server_long_query_time은 데이터베이스에 대한 진단 항목 중 하나인 슬로우 쿼리(slow query)를 판단하기 위한 기준 시간을 설정하는 파라미터이다. 디폴트 값은 10(sec)이며, 이 경우 실행 시간이 10초를 초과하는 질의를 슬로우 쿼리로 판단한다. server_long_query_time 파라미터는 CUBRID 매니저를 통한 데이터베이스 진단 기능이 활성화된 경우에 적용되므로, execute_diag 파라미터가 ON으로 설정된 상태에서만 효력이 있다.
