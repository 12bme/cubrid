질의 자동화

데이터베이스에 로그인하면, 접근할 수 있는 질의 자동화를 탐색 트리에서 볼 수 있다.

CUBRID 매니저 서버를 통해서 주기적으로 질의를 수행하고자 할 경우 질의 자동화를 사용한다.

질의 자동화 추가

  • 질의 ID: 질의 작업의 이름을 설정한다. 질의 작업이 여러 개일 수 있으므로, 하나의 데이터베이스 내에서는 유일해야 한다.
  • 실행 주기: 질의를 자동 실행할 주기를 매월, 매주, 매일, 특정일 중에서 선택할 수 있다.
  • 상세 주기: 선택한 실행 주기를 상세하게 설정할 수 있다.
  • 실행 시간: 질의를 자동 실행할 시간을 입력한다. 시간과 분을 각각 입력한다.
  • 질의 구문: 자동 실행할 질의문을 입력한다. 등록된 질의문은 설정된 시간마다 주기적으로 자동 실행되지만, 실행 결과는 기록되지 않음을 유의한다.

자동 실행되는 질의의 결과는 기록되지 않으므로, SELECT 문을 자동 실행하는 것은 아무 의미가 없다. 그러나, 통계 정보의 갱신(UPDATE STATISTICS)이나, 주기적인 데이터 갱신을 하고자 할 경우에는 질의 자동화 기능이 유용하다. 자동 실행한 질의의 결과는 $CUBRID/log/manager/auto_execquery.log 파일에 기록된다.

질의 자동화 로그

질의 자동화에서 발생한 로그를 제공한다. 로그는 성공 실패에 상관없이 모두 남는다.

질의 자동화 편집 / 삭제

등록된 질의 자동화 작업을 질의 자동화 추가 시와 동일한 방법으로 편집하거나, 필요없는 질의 자동화 작업을 삭제한다.

탐색 트리에서 해당 질의 자동화를 마우스 오른쪽 버튼 클릭하여 [질의 자동화 편집] 또는 [질의 자동화 삭제]를 선택한다.
