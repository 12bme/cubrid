매뉴얼 구성

CUBRID 데이터베이스 관리 시스템(Database Management System, DBMS) 매뉴얼의 구성은 다음과 같다.

  • CUBRID 시작 : CUBRID를 처음 시작하는데 참고할 수 있는 내용을 제공한다. 시스템 설치 및 실행 방법, CSQL 인터프리터와 CUBRID 매니저의 간단한 사용 방법을 찾아볼 수 있다. 또한, JDBC나 PHP, ODBC, CCI 등을 이용한 응용 프로그램 작성 예제를 제공한다.
  • CUBRID 소개 : CUBRID 데이터베이스 관리 시스템의 구조 및 특징을 설명한다.
  • CSQL 인터프리터 : CSQL은 CUBRID에서 명령어 방식으로 SQL 문을 사용할 수 있는 프로그램이다. CSQL 인터프리터의 간단한 사용법과 관련 명령어들을 설명한다.
  • 관리자 안내서 : 데이터베이스 생성 및 삭제, 백업 및 복구, 마이그레이션, 복제 작업 등을 수행하는 방법을 설명한다. CUBRID 서버, 브로커 및 매니저 서버 등의 구동과 정지 등을 수행하는 CUBRID 유틸리티의 사용법도 포함되어 있다.
  • 성능 튜닝 : 성능에 영향을 미칠 수 있는 시스템 파라미터의 설정 방법을 설명한다. 서버, 브로커, CUBRID 매니저 서버 각각의 설정 파일 사용법과 개별 파라미터의 의미를 설명한다.
  • CUBRID SQL 설명서 : 데이터 타입, 함수와 연산자, 데이터 조회나 테이블 조작 등, CUBRID에서 사용할 수 있는 SQL 구문에 대해 설명한다. 인덱스나 트리거, 분할, 시리얼 및 사용자 정보 변경 등의 작업을 위한 SQL 구문도 찾아볼 수 있다.
  • CUBRID 매니저 : GUI (Graphic User Interface) 방식의 데이터베이스 관리 및 질의 도구인 "CUBRID 매니저"의 사용 방법을 설명한다. "CUBRID 매니저"는 다양한 관리 작업을 손쉽게 하도록 도와주며, 접속한 데이터베이스에 SQL 문을 실행할 수 있는 "질의 편집기" 기능을 제공한다.
  • API 레퍼런스 : JDBC API, ODBC API, OLE DB API, PHP API 및 CCI API에 대해 설명한다.
매뉴얼 규약

다음 표는 CUBRID 데이터베이스 관리 시스템 제품 매뉴얼에서 '문장', '명령어', 그리고 '텍스트 안에서의 참조'를 식별하는데 사용되는 문서 규약이다.

  규약

설명

예제

Italics

Italics type은 변수 이름을 나타내는데 사용된다.

persistent:

stringVariableName

Boldface

Boldface는 멤버 함수 이름, 클래스 이름, 상수, CUBRID 키워드 또는 다른 필수 문자와 같은 이름을 나타내는데 사용된다.

fetch ( ) member function

class odb_User

Constant Width

Constant Width type의 글꼴은 코드 예제 또는 명령어의 실행 및 결과를 나타내는데 사용된다.

csql database_name

UPPER-CASE

UPPER-CASE는 CUBRID 키워드를 나타내는데 사용된다(Boldface 참조).

SELECT

Single Quotes (' ')

중괄호, 대괄호와 같이 사용하는 작은 따옴표(' ')는 문법의 필요한 부분을 나타낸다. 또한 작은 따옴표는 스트링을 싸는데도 사용된다.

{'{' const_list'}'}

Brackets ([ ])

대괄호([  ])는 파라미터나 키워드가 선택적임을 나타내는데 사용된다.

[ONLY]

Underline( _ )

밑줄(_)은 명시되지 않을 때의  디폴트 키워드를 나타내는데 사용된다.

[DISTINCT|UNIQUE|ALL]

Vertical bar( | )

수직 막대(|)는 여럿 중의 하나만을 선택할 수 있음을 나타내는데 사용된다.

[ATTRIBUTE|COLUMN]

Braces around parameters({  })

중괄호로 싼 파라미터는 문장의 문법으로 여럿 중에서 하나만을 반드시 선택해야 함을 나타내는데 사용된다.

CREATE {CLASS|TABLE}

Braces around values({ })

중괄호로 싼 값은 집합을 구성하는 원소를 나타내는데 사용된다.

{2. 4, 6}

Braces with ellipsis({ }...)

중괄호 뒤의 생략 부호(...)는 파라미터 지정이 반복될 수 있음을 나타내는데 사용된다.

{, class_name}...

Angle brackets(< > )

산괄호(< > )는 단일 키 또는 일련의 키 입력을  나타내는데 사용된다. 

< Ctrl+n>
