CUBRID에서 더티 인스턴스(dirty instances)를 다루는 방법

CUBRID는 다양한 상황에서 워크스페이스(클라이언트)의 더티 인스턴스를 데이터베이스(서버)로 출력한다(다른 경우에도 더티 인스턴스가 출력될 수 있다).

  • 모든 더티 인스턴스는 커밋되는 동안에 출력된다.
  • 워크스페이스가 많은 인스턴스를 적재할 때 일부 더티 인스턴스는 출력된다.
  • 클래스의 더티 인스턴스는 클래스가 갱신될 때(스키마가 갱신될 때) 출력된다.
  • 클래스의 더티 인스턴스는 클래스가 질의될 때 출력된다.
  • 더티 인스턴스의 일부는 서버 함수가 호출될 때 출력될 수 있다.
