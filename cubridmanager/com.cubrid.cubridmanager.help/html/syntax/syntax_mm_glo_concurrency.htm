동시성

동시성 규칙은 다른 CUBRID 클래스 인스턴스에 적용되는 것과 같이 glo 인스턴스에도 동일하게 적용된다. glo 인스턴스에서 읽기 잠금을 공유함으로써 동시에 여러 사용자가 glo 데이터를 읽을 수 있다. 해당 glo 인스턴스에 다른 사용자가 쓰기 잠금을 가지고 있지 않으면, 읽기 잠금을 부여할 수 있다. 특정 인스턴스에 읽기 잠금이나 쓰기 잠금이 없다면, 트랜잭션이 수행되는 동안 glo 데이터에 대해 한 사용자에 의해서만 갱신에 대한 쓰기 잠금이 허용된다.

glo에 대한 읽기와 쓰기 잠금은 모든 glo에 적용된다. glo 데이터의 부분 잠금은 지원하지 않는다.

FBO에는 한가지 특별한 동시성 조건이 적용된다. CUBRID 데이터베이스 시스템은 동일한 파일을 참조하는 glo 인스턴스를 생성하는 것을 허용한다.

모든 glo 인스턴스가 같은 경로명을 사용한다면, 같은 동시성 법칙이 적용된다. 그러나 절대 경로명을 사용하는 glo 인스턴스와 상대 경로명을 사용하는 glo 인스턴스가 같은 파일을 참조한다면, 두 인스턴스에 대해 일관적인 갱신 접근을 보장할 수 없다. 마찬가지로, 절대 경로명을 사용하는 glo 인스턴스와 연결(link)를 사용하는 glo 인스턴스가 같은 파일을 참조하는 경우에도 일관적인 갱신 접근을 보장할 수 없다.

CUBRID는 데이터베이스를 통해 FBO에 간접적으로 접근하기 위해서 동시성 제어를 지원한다. 그러나 외부 사용자나, 응용 프로그램에 의한 FBO의 직접적인 접근(또는 갱신)은 막을 수 없다.
