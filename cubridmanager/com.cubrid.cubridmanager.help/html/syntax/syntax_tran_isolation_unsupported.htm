지원하지 않는 격리 수준 조합

비록 SET TRANSACTION ISOLATION LEVEL 구문이 다른 조합을 정의할 수 있도록 하더라도 아래의 조합은 허용하지 않는다. 시스템은 경고 메시지를 주고 명시된 것에 가까운 격리 수준을 선택한다. 아래는 허용하지 않는 격리 수준이다.

  • READ COMMITTED CLASS with REPEATABLE READ INSTANCES
  • READ UNCOMMITTED CLASS with REPEATABLE READ INSTANCES

만약 클래스의 구조가 갱신되면 클래스의 인스턴스에 대한 읽기는 반복 불가능하다. 그러므로 위 조합은 적합하지 않다.

  • READ UNCOMMITTED CLASS with READ COMMITTED INSTANCES
  • READ UNCOMMITTED CLASS with READ UNCOMMITTED INSTANCES

위 격리 수준은 잘 이해되지 않으며 시스템은 실패할 수 있다. 예를 들어 한 트랜잭션에서 질의가 수행되고 질의의 중간에 동일한 클래스가 다른 트랜잭션에 의해 삭제되면 많은 문제가 발생할 것이다.
